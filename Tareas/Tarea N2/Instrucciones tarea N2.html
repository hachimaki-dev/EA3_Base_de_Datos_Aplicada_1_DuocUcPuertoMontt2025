Conceptos Básicos: DDL y DML

DDL (Data Definition Language)

DDL son los comandos que definen la estructura de la base de datos. Crean, modifican o eliminan objetos como tablas.

En este taller usarás: CREATE TABLE, PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE

Ejemplo: CREATE TABLE usuarios (id NUMBER PRIMARY KEY, nombre VARCHAR2(100) NOT NULL);

DML (Data Manipulation Language)

DML son los comandos que manipulan los datos dentro de las tablas. Insertan, actualizan, eliminan o consultan información.

En este taller usarás: INSERT, UPDATE, DELETE, SELECT

Ejemplo: INSERT INTO usuarios (id, nombre) VALUES (1, 'María García');

¿Qué es la Tercera Forma Normal (3FN)?

La 3FN es un estándar de diseño que garantiza que tu base de datos esté bien organizada y sin redundancia.

Una tabla está en 3FN cuando cumple:

  • Cada campo depende de la llave primaria (no hay datos repetidos innecesariamente)
  • No hay dependencias transitivas (un campo no debe depender de otro campo que no sea la llave primaria)
  • Cada tabla representa un solo concepto o entidad

Ejemplo práctico:

❌ MAL: Una tabla "Pedidos" que incluya: id_pedido, fecha, id_cliente, nombre_cliente, dirección_cliente

✅ BIEN: Dos tablas: "Pedidos" (id_pedido, fecha, id_cliente) y "Clientes" (id_cliente, nombre, dirección)

Piensa: "¿Este dato se va a repetir muchas veces? Si sí, probablemente necesita su propia tabla."

Tarea 3: Script DDL (CREATE TABLE)

¿Qué debes escribir aquí?

Código SQL completo con los comandos CREATE TABLE para TODAS tus tablas. Incluye:

  • PRIMARY KEY en cada tabla
  • FOREIGN KEY donde corresponda
  • Restricciones NOT NULL, UNIQUE según tu diseño

Ejemplo de sintaxis:

CREATE TABLE usuarios (
  id_usuario NUMBER PRIMARY KEY,
  nombre VARCHAR2(100) NOT NULL,
  email VARCHAR2(100) UNIQUE NOT NULL
);

CREATE TABLE publicaciones (
  id_publicacion NUMBER PRIMARY KEY,
  id_usuario NUMBER NOT NULL,
  contenido VARCHAR2(500),
  fecha DATE NOT NULL,
  FOREIGN KEY (id_usuario) REFERENCES usuarios(id_usuario)
);

Importante: Crea primero las tablas "padre" (las que no tienen FK) y luego las tablas "hijo" (las que sí tienen FK).

Comandos UPDATE para modificar datos que ya existen en tus tablas.

Requisito mínimo: 2 UPDATE diferentes.

¡SIEMPRE usa WHERE! Si no pones WHERE, modificarás TODOS los registros de la tabla.

Tarea 7: Scripts SELECT

¿Qué debes escribir aquí?

Comandos SELECT para consultar y mostrar información de tus tablas.

Requisito mínimo: 4 consultas SELECT diferentes.

Ejemplos de consultas que puedes hacer:

  • Mostrar todos los registros de una tabla
  • Mostrar registros que cumplan una condición específica (WHERE)
  • Mostrar solo algunos campos específicos
  • Ordenar los resultados (ORDER BY)

Sintaxis básica:

-- Consulta 1: Todos los usuarios
SELECT * FROM usuarios;

-- Consulta 2: Publicaciones de un usuario específico
SELECT * FROM publicaciones WHERE id_usuario = 1;

-- Consulta 3: Solo nombres y emails
SELECT nombre, email FROM usuarios;

-- Consulta 4: Likes ordenados por fecha
SELECT * FROM likes ORDER BY fecha DESC;

Tip: Escribe consultas útiles que mostrarían información relevante de tu funcionalidad.

Preguntas Frecuentes (FAQ)

❓ ¿Puedo elegir cualquier app?

Sí, pero debe ser una app que tu profesor apruebe. Apps conocidas como Spotify, Instagram, WhatsApp, Netflix, Uber, TikTok, Twitter/X, YouTube, son perfectas. Si quieres elegir una menos conocida, pregunta primero.

❓ ¿Qué significa "una funcionalidad específica"?

Significa UNA característica concreta, NO toda la app. Ejemplos: "dar like", "comentar", "crear playlist", "dar match", "calificar conductor", "agregar a favoritos". NO es válido decir "toda Instagram" o "el sistema de usuarios".

❓ ¿Cuántas tablas debo crear?

Depende de tu funcionalidad, pero típicamente entre 2 y 5 tablas. Si tienes solo 1 tabla, probablemente tu modelo no está en 3FN. Si tienes más de 6, probablemente estás modelando demasiado.

❓ ¿Mis respuestas se pierden si cierro el navegador?

❓ ¿Tengo que ejecutar el código SQL en Oracle?

Eso depende de las instrucciones de tu profesor. Este taller NO ejecuta el código automáticamente. Solo lo guardas aquí para escribirlo. Pregunta a tu profesor si también debes ejecutarlo en Oracle SQL Developer o similar.

❓ ¿Qué pasa si mi código SQL tiene errores?

Este taller NO verifica errores de sintaxis. Tú eres responsable de revisar tu código. Usa Oracle SQL Developer para probarlo si tienes dudas. Revisa los ejemplos de sintaxis en cada tarea.

❓ ¿Puedo usar tildes y eñes en los nombres de tablas/campos?

Técnicamente sí, pero NO es recomendable. Usa solo letras sin tilde, números y guión bajo. Ejemplo: usa "usuarios" no "usuários", usa "nombre" no "nombré".

❓ ¿Qué tipo de dato uso para fechas?

Usa DATE para fechas en Oracle. Ejemplo: fecha_creacion DATE

❓ ¿Qué tipo de dato uso para textos?

Usa VARCHAR2(tamaño). Ejemplo: nombre VARCHAR2(100), email VARCHAR2(150). El número entre paréntesis es el máximo de caracteres.

❓ ¿Qué tipo de dato uso para números?

Usa NUMBER. Para números enteros: NUMBER. Para decimales: NUMBER(10,2) donde 10 es total de dígitos y 2 son decimales.

❓ ¿Cómo sé si mi modelo está en 3FN?

Pregúntate: "¿Tengo datos que se repiten en múltiples filas?" Si sí, probablemente NO está en 3FN. Ejemplo: si guardas el nombre del usuario en cada like que da, está mal. El nombre debe estar solo en la tabla USUARIOS.

❓ ¿Puedo poner más de una PRIMARY KEY en una tabla?

NO. Cada tabla tiene UNA sola PRIMARY KEY. Puede ser un solo campo o una combinación de campos (PK compuesta), pero se declara una sola vez.

❓ ¿Qué orden uso para crear las tablas?

Primero crea las tablas que NO tienen FOREIGN KEY (tablas "padre"). Después crea las tablas que SÍ tienen FOREIGN KEY (tablas "hijo"). Ejemplo: primero USUARIOS, luego PUBLICACIONES (que referencia a USUARIOS).

❓ ¿Tengo que poner punto y coma (;) al final de cada comando SQL?

SÍ. Cada comando SQL debe terminar con punto y coma (;). Es obligatorio en Oracle.

❓ ¿Importa si escribo en mayúsculas o minúsculas?

Los comandos SQL (SELECT, CREATE, INSERT) tradicionalmente se escriben en MAYÚSCULAS, pero Oracle los acepta en minúsculas también. Los nombres de tablas y campos puedes escribirlos como quieras, pero sé consistente.

Usa el botón "Borrar respuesta" de esa tarea. Esto limpiará el textarea y eliminará lo guardado. Luego puedes escribir de nuevo.

❓ ¿Puedo copiar y pegar código de internet?
