Taller: Modelado de Base de Datos para Apps Reales

Modelado Físico en 3FN – Oracle SQL 23c
Duración estimada: 60 minutos.

Cómo usar este taller

  • Trabaja en equipo.
  • Completa cada tarea en orden.
  • Escribe tus respuestas en los cuadros del taller.
  • Utiliza los botones de Guardar y Borrar.
  • Las respuestas quedan almacenadas en el navegador (localStorage).
  • Revisa el checklist final antes de entregar.

Qué vas a hacer

  • Elegir una app famosa.
  • Identificar una funcionalidad específica.
  • Diseñar un modelo físico de base de datos.
  • Construir SQL DDL y DML para esa funcionalidad.

Conceptos clave

DDL (Data Definition Language)

Define la estructura de la base de datos. Usarás:

  • CREATE TABLE
  • PRIMARY KEY
  • FOREIGN KEY
  • UNIQUE
  • NOT NULL

DML (Data Manipulation Language)

Manipula datos. Usarás:

  • INSERT
  • UPDATE
  • DELETE
  • SELECT

3FN (Tercera Forma Normal)

  • Evita redundancias.
  • Toda columna depende directamente de la clave primaria.
  • Sin dependencias transitivas.
  • Una tabla representa una sola entidad.

Tarea 1: App y Funcionalidad

  • Funcionalidad específica
  • Descripción en 2–3 frases

Tarea 2: Modelo Físico

Lista tablas, campos, tipos, PK, FK y restricciones.

Tarea 3: Script DDL

Crea todas las tablas:

  • Primero las tablas padre.
  • Luego las tablas hijo (con FK).

Tarea 4: INSERT

Agrega 3 registros por tabla (primero las tablas padre).

Tarea 5: UPDATE

Crea al menos 2 operaciones UPDATE con sentido dentro de la lógica de la app.
